# Governance Loops

## Monthly Loop

Use the monthly loop for normal review of:

- architecture changes
- business capability changes
- module changes
- solution-pack updates
- documentation and structure updates

## Weekly Loop

Use the weekly loop only for critical items:

- vulnerabilities
- patches
- security advisories
- provider changes
- certificate expiration
- secrets exposure
- runtime health

## Rules

- No uncontrolled recursive automation.
- Controller-gated review lifecycle.
- Human approval required before production changes.
- All changes must be explainable and auditable.
