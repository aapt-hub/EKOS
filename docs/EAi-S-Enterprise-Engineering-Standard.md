# EAi-S Enterprise Engineering Standard

EAi-S defines the standard engineering rules for the EAi platform.

## Standard Rules

- Use documentation-first and structure-first work where runtime is not yet approved.
- Freeze architecture before kernel implementation begins.
- Keep `EGK`, `ECK`, `EDTi`, and `ESF` as the owning kernel hierarchy.
- Keep platform products as consumers of architecture.
- Keep business modules modular.
- Keep solution packs optional, certified, and composable.
