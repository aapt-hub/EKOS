# Kernel Contract Boundary

The Kernel owns only contracts.

The Kernel owns:

- Contracts.
- Schemas.
- Registries.
- Validation.
- Discovery.

The Kernel never owns:

- Business logic.
- Runtime behavior.
- Services.
- APIs.
- Databases.
- Endpoint agents.
- Workflow execution.
- Module implementations.

## Canonical Contract Surface

Every Enterprise Asset contract must include identity, relationships, policies, capabilities, events, telemetry, lifecycle, governance, and intelligence.

## Validation Note

A Kernel contract is valid only when it uses canonical vocabulary, references Kernel schemas, references Kernel registries, and remains implementation-free.
